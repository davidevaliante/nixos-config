# Smoke tests

Run after `sudo nixos-rebuild switch --flake .#nixos --no-update-lock-file`,
log out, and pick **Hyprland** or **Niri** from the GDM gear menu.

Most checks are commands you run in **kitty** (`Super+Return` after login).
Visual checks are marked with `[eye]`.

---

## After login (any compositor)

```sh
echo $XDG_CURRENT_DESKTOP    # → Hyprland or niri
echo $XDG_SESSION_TYPE       # → wayland
echo $WAYLAND_DISPLAY        # → wayland-1 (non-empty)
loginctl show-session $XDG_SESSION_ID -p Type    # → Type=wayland
```

`[eye]` Top bar present (waybar): workspace numbers on the left,
clock centered, tray + volume + network + battery + notification icon on the right.

`[eye]` Background is the gradient generated by stylix
(roughly catppuccin-mocha colors).

`[eye]` Window borders + rounded corners + slight blur on translucent windows.

## Theme / fonts

```sh
fc-match monospace                  # → JetBrainsMonoNerdFont-Regular...
fc-match sans-serif                 # → Inter-Regular...
gsettings get org.gnome.desktop.interface gtk-theme   # → some catppuccin variant
gsettings get org.gnome.desktop.interface cursor-theme # → Bibata-Modern-Classic
echo "  test"                  # ← Nerd Font icons should render, not boxes
```

`[eye]` `kitty` window has rounded corners, no system titlebar, slight transparency.

## Notifications (swaync)

```sh
notify-send "Hello" "from swaync"
notify-send -u critical "Critical" "this should be styled differently"
swaync-client --count          # → number of pending notifications
```

`Super+N` → control center panel slides in from the right with notification
history + DnD toggle. The bell icon in waybar's top-right reflects state.

## Clipboard history (cliphist + wl-clipboard)

```sh
echo "first"  | wl-copy
echo "second" | wl-copy
wl-paste                       # → second
cliphist list                  # → both entries (stored by the watcher)
```

If `cliphist list` is empty, the spawn-at-startup watcher didn't start —
restart it with `wl-paste --type text --watch cliphist store &` and re-test.

## Hyprland-specific

```sh
hyprctl version                # build info
hyprctl monitors               # outputs + active workspace
hyprctl clients                # all current windows
hyprctl activeworkspace
hyprctl getoption decoration:rounding   # → 8
```

Extra binds:
- `Super+V` toggle floating
- `Super+F` fullscreen
- `Super+P` pseudo (dwindle layout)
- `Super+J` toggle split direction
- `Super+0` workspace 10
- Hold `Super` and **drag** with left/right mouse → move/resize floating window.

## Niri-specific

```sh
niri msg version
niri msg outputs
niri msg workspaces
niri msg focused-window
niri validate                  # config sanity check
```

Niri's columns scroll horizontally — open 3+ windows on the same workspace
to see it. Extra binds:
- `Super+R` cycle preset column widths (1/3 → 1/2 → 2/3)
- `Super+F` maximize column
- `Super+Shift+F` true fullscreen
- `Super+W` toggle tabbed display in current column
- `Super+V` toggle floating


## If something looks wrong

```sh
journalctl --user -b -p warning      # warnings + errors from user services this boot
journalctl -b -p err                 # system errors this boot
hyprctl logs                         # Hyprland's runtime log (Hyprland session)
niri msg                             # check niri ipc is responsive (niri session)
```

Common things to check:
- Waybar missing → `systemctl --user restart waybar.service`
- Notifications missing → `systemctl --user restart swaync.service`
- Theme didn't apply to a GTK app → that app may need to be relaunched after the rebuild.
- Cursor is the default arrow → log out / log back in, the cursor theme env vars only get set at session start.
